# Шаблон для Python проектов

Этот репозиторий представляет собой шаблон для создания Python проектов с предварительно настроенной средой разработки.

## Описание

Шаблон включает в себя настроенные инструменты для форматирования кода, управления зависимостями и конфигурацию
редактора кода VS Code. Он предназначен для быстрого старта разработки Python приложений с соблюдением лучших практик.

## Требования

- Python 3.13.5 или выше
- [uv](https://docs.astral.sh/uv/) для управления зависимостями
- [Ruff](https://docs.astral.sh/ruff/) для форматирования и линтинга кода

## Установка и настройка

1. Клонируйте репозиторий:

   ```bash
   git clone <url-репозитория>
   cd Template-for-Python-Project
   ```

   или создаете свою папку с проектом

   ```bash
   mkdir My-Python-Project
   cd My-Python-Project
   git clone <url-репозитория> .
   ```

   > Точка `.` указывает на то, что репозиторий будет клонирован в текущую папку.

   **Команда Powershell (Windows):**

   ```powershell
   git clone git@github.com:Hetvild/Template-for-Python-Project.git . ; Remove-Item -Recurse -Force .git && git init
   ```

   **Команда Bash (Linux/Mac):**

   ```powershell
   git clone git@github.com:Hetvild/Template-for-Python-Project.git . && rm -rf .git && git init
   ```

2. Установите зависимости с помощью `uv`:

   ```bash
   uv sync
   ```

3. Активируйте виртуальное окружение:

   ```bash
   uv run python src/main.py
   ```

## Структура проекта

```text
Template-for-Python-Project/
├── .vscode/             # Настройки VS Code
│   └── settings.json    # Конфигурация редактора
├── src/                 # Исходный код приложения
│   ├── core 
│   └── main.py          # Точка входа в приложение
├── pyproject.toml       # Конфигурация проекта и зависимостей
├── .gitignore           # .gitignore файл для Git
├── .dockerignore        # .dockerignore файл для сборки проектов в Docker
├── .python-version      # Версия Python которая используется в проекте
└── README.md            # Документация проекта
```

## Конфигурация

### Python и зависимости

Проект использует Python 3.13.5.
Управление зависимостями осуществляется через `uv` с использованием файла `pyproject.toml`.

### Форматирование кода

Форматирование кода настроено с помощью `Ruff` со следующими параметрами:

- Длина строки: 88 символов
- Ширина отступа: 4 пробела
- Стиль кавычек: двойные кавычки
- Форматирование docstring: включено

### Настройки VS Code

Редактор VS Code настроен на использование:

- Шрифта: JetBrains Mono
- Размера шрифта: 13
- Длины строки: 88 символов (показывается линейкой)
- Автоматического форматирования при сохранении
